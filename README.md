# 동네 농구 매칭 - 용어 사전(유비쿼터스 언어 정리)
유비쿼터스 언어는 팀원들의 '협업'과 '의사소통'의 중심이 됩니다.
> **중요**: 이 문서는 기획자, PM, 프론트엔드, 백엔드 개발자가 **모두 같은 언어로 대화**하기 위한 것입니다.</br>
> 기술 용어를 최소화하고, 비즈니스 관점에서 쉽게 이해할 수 있도록 작성되었습니다.

---

## 🔴 Match Context (경기 관련)

### 경기 (Match)

| 한글명 | 영문명 | 설명 |
|--------|--------|------|
| 경기 | Match | 특정 날짜, 시간, 장소에서 하는 농구 경기 |
| 주최자 | Host | 경기를 만들고 관리하는 사람 |
| 참여자 | Participant | 경기에 확정된 사람 |
| 코트 | Court | 경기를 할 농구 코트 |
| 시간대 | Time Slot | 경기를 할 시간대 (예: 내일 오후 7시~9시) |
| 최대 참여자 | Max Participants | 한 경기에 참여할 수 있는 최대 인원 |
| 현재 참여자 | Current Participants | 현재 경기에 신청/확정된 참여자 수 |

### 경기 상태 (Match Status)

| 상태명 | 영문명 | 설명 | 다음 상태 |
|--------|--------|------|----------|
| 모집 중 | RECRUITING | 인원이 부족해서 아직 신청을 받는 중 | CONFIRMED |
| 확정됨 | CONFIRMED | n명이 모두 모인 경기 (신청 불가) | COMPLETED |
| 완료됨 | COMPLETED | 경기 시간이 지나 끝난 상태 | - |
| 취소됨 | CANCELLED | Host가 경기를 취소한 상태 | - |

### 경기의 규칙과 제약

| 규칙 | 설명 |
|------|------|
| **정원** | 최대 n명 (변경 불가) |
| **Host 필수** | 경기마다 Host는 반드시 1명 필요 |
| **자동 포함** | Host가 경기 생성 → Host 자동 포함 |
| **자동 확정** | n명이 모이면 경기 상태 자동 변경: RECRUITING → CONFIRMED |
| **코트 중복 불가** | 같은 시간에 같은 코트에서는 1경기만 가능 |
| **Host 신뢰도** | 신뢰도 {a} 이상이어야 경기 생성 가능 |

## Participation Context (신청 관련)

### 신청 (Participation)

| 한글명 | 영문명 | 설명 |
|--------|--------|------|
| 신청 | Participation | 사용자가 어떤 경기에 참가하고 싶다는 의사 표현 |
| 신청자 | Applicant | 특정 경기에 신청한 사용자 |
| 신청 대상 | Target | Applicant가 신청한 Match (경기) |
| 신청 시간 | Applied At | 신청을 한 시각 |

### 신청 상태 (Participation Status)

| 상태명 | 영문명 | 설명 | 다음 상태 |
|--------|--------|------|----------|
| 신청함 | APPLIED | 경기에 신청했지만 아직 확정 안 됨 | CONFIRMED |
| 확정됨 | CONFIRMED | 경기 인원 n명이 찬 후 자동 확정 | COMPLETED / NO_SHOW |
| 완료됨 | COMPLETED | 경기가 정상적으로 끝남 | - |
| 노쇼 | NO_SHOW | 확정된 경기에 나타나지 않음 | - |
| 취소됨 | CANCELLED | 신청 취소 또는 경기 자체 취소 | - |

### 신청의 규칙과 제약

| 규칙 | 설명 |
|------|------|
| **중복 신청 불가** | 한 사용자가 같은 경기에 2번 신청 불가 |
| **시간 겹침 불가** | 한 사용자가 겹치는 시간대에 여러 경기 신청 불가 |
| **자동 상태 변경** | 경기가 CONFIRMED되면 신청도 자동으로 CONFIRMED |
| **경기마다 다른 신청자** | 같은 사람도 경기 B에 신청하면 경기 B의 신청자가 됨 |

## User Context (사용자 관련)

### 사용자 (User)

| 한글명 | 영문명 | 설명 |
|--------|--------|------|
| 사용자 | User | 서비스에 가입한 개인 |
| 프로필 | Profile | 사용자의 기본 정보 (이름, 전화, 이메일, 사진) |
| 이름 | Name | 사용자의 이름 |
| 전화번호 | Phone | 사용자의 전화번호 |
| 이메일 | Email | 사용자의 이메일 주소 |
| 프로필 사진 | Profile Image | 사용자의 프로필 사진 |

### 신뢰도 (Reputation)

| 한글명 | 영문명 | 설명 |
|--------|--------|------|
| 신뢰도 | Reputation | 다른 사람들이 평가한 사용자의 신뢰성 점수 |
| 신뢰도 점수 | Reputation Score | 평점들의 평균 (1.0 ~ 5.0) |
| 평점 | Rating | 경기 후 상대방이 준 별점 (1~5점) |
| 리뷰 | Review | 경기 후 상대방이 쓴 평가 의견 |
| 노쇼 기록 | No-Show Record | 확정된 경기에 나타나지 않은 기록 |
| 노쇼 횟수 | No-Show Count | 누적 노쇼 횟수 |

### 신뢰도 점수 (Reputation Score)

| 점수 | 설명 | 상태 | 영향 |
|------|------|------|------|
| 5.0 | 매우 높음 | ⭐⭐⭐⭐⭐ | 모두가 이 사람의 경기에 신청하고 싶어함 |
| 4.0 ~ 4.9 | 높음 | ⭐⭐⭐⭐ | 좋은 평가 (경기 쉽게 참여 가능) |
| 3.0 ~ 3.9 | 중간 | ⭐⭐⭐ | 보통 수준 (기능 제약 없음) |
| 2.0 ~ 2.9 | 낮음 | ⭐⭐ | ⚠️ 경고 상태 (일부 기능 제한 시작) |
| 0.0 ~ 1.9 | 매우 낮음 | ⭐ | 🚫 자동 정지 직전 |

### 신뢰도 변동

| 상황 | 변동 | 설명 |
|------|------|------|
| **경기 정상 완료** | +0.2 | 경기에 참석해서 완료 |
| **높은 평점** | +0.3 | 상대방이 5점 주기 |
| **좋은 리뷰** | +0.1 | 긍정적인 평가 받기 |
| **노쇼** | -0.5 | 확정된 경기에 나타나지 않음 (심각) |
| **낮은 평점** | -0.3 | 상대방이 1~2점 주기 |
| **안 좋은 리뷰** | -0.1 | 부정적인 평가 받기 |

### 노쇼 (No-Show)

| 한글명 | 영문명 | 설명 |
|--------|--------|------|
| 노쇼 | No-Show | 확정된 경기에 나타나지 않은 것 |

### 노쇼 기록되는 조건

| 조건 | 설명 |
|------|------|
| **경기 상태** | = CONFIRMED (확정됨) |
| **경기 시간** | 이미 지남 |
| **참석 여부** | 사용자가 나타나지 않음 |

### 노쇼의 결과

| 결과 | 설명 |
|------|------|
| **신뢰도 감소** | -0.5점 (심각한 감소) |
| **기록 누적** | No-Show Count 증가 |
| **3회 이상** | 자동 정지 (경기 생성/신청 불가) |

## 🟡 Location Context (장소 관련)

### 코트 (Court)

| 한글명 | 영문명 | 설명 |
|--------|--------|------|
| 코트 | Court | 농구를 할 수 있는 실제 장소 |
| 코트 이름 | Name | 코트의 이름 (예: "광주 시청 농구장") |
| 위치 | Location | 지도상의 정확한 위치 (위도, 경도) |
| 위도 | Latitude | 위도 좌표 |
| 경도 | Longitude | 경도 좌표 |
| 시설 정보 | Facility Info | 실내/실외, 조명 여부, 주차장, 탈의실 등 |
| 실내/실외 | Indoor/Outdoor | 코트가 실내인지 실외인지 |
| 조명 | Lighting | 야간 경기 가능 여부 |
| 주차장 | Parking | 주차장 있는지 여부 |
| 탈의실 | Changing Room | 탈의실 있는지 여부 |

### Match Status (경기 상태)

```
┌─────────────────────────────────────────────────────┐
│                  경기 생성                           │
└────────────────┬────────────────────────────────────┘
                 │
                 ▼
        ┌────────────────┐
        │  RECRUITING    │ (모집 중)
        │   (모집 중)    │ → 신청 가능
        └────────┬───────┘
                 │ (4명 모이면 자동)
                 ▼
        ┌────────────────┐
        │   CONFIRMED    │ (확정됨)
        │   (확정됨)     │ → 신청 불가능
        └────────┬───────┘
                 │ (경기 시간 지나면)
                 ▼
        ┌────────────────┐
        │   COMPLETED    │ (완료됨)
        │   (완료됨)     │ → 평가 가능
        └────────────────┘

    또는 언제든:
        ┌────────────────┐
        │   CANCELLED    │ (취소됨)
        │   (취소됨)     │
        └────────────────┘
```

### Participation Status (신청 상태)

```
┌─────────────────────────────────────────────────────┐
│              사용자가 신청                           │
└────────────────┬────────────────────────────────────┘
                 │
                 ▼
        ┌────────────────┐
        │    APPLIED     │ (신청함)
        │   (신청함)     │ → Host는 알림 받음
        └────────┬───────┘
                 │ (경기가 4명 모이면 자동)
                 ▼
        ┌────────────────┐
        │   CONFIRMED    │ (확정됨)
        │   (확정됨)     │ → 모두 알림 받음
        └────────┬───────┘
                 │ (경기 시간 지나면)
                 │
         ┌───────┴───────┐
         │               │
         ▼               ▼
    ┌─────────┐     ┌──────────┐
    │COMPLETED│     │ NO_SHOW  │
    │(완료됨) │     │ (노쇼)   │
    └─────────┘     │신뢰도↓  │
                    └──────────┘

    또는 언제든:
        ┌────────────────┐
        │   CANCELLED    │ (취소됨)
        │   (취소됨)     │
        └────────────────┘
```

## 🎯 자주 헷갈리는 용어 정리

### Match vs Participation

| 항목 | Match (경기) | Participation (신청) |
|------|-----------|---------|
| **정의** | 특정 날짜, 시간, 장소에서 하는 경기 | 사용자가 경기에 참가하려는 의사 |
| **주인** | Host | Applicant |
| **개수** | 1개 | 여러 개 (신청한 사람만큼) |
| **상태 변경** | Host나 시간에 따라 | 경기가 확정되면 자동 변경 |
| **예시** | "경기 1: 내일 오후 7시 광주 코트" | "철수의 신청: 경기 1에 신청함" |

### Host vs Applicant vs Participant

| 항목 | Host | Applicant | Participant |
|------|------|-----------|-------------|
| **역할** | 경기를 만드는 사람 | 경기에 신청한 사람 | 경기에 확정된 사람 |
| **타이밍** | 경기 생성 시 | 신청 시 | 경기 확정 시 |
| **상태** | 항상 CONFIRMED | APPLIED → CONFIRMED | CONFIRMED (4명) |
| **예시** | "내가 경기를 만들었어" | "철수가 신청했어" | "민혁, 철수, 영희, 준호 4명이 경기해" |

### Status (상태) vs State (상태)

> 같은 의미이므로 혼동 없이 "Status"로 통일

| 항목 | 정의 |
|------|------|
| **Status** | 현재의 상태 (예: RECRUITING, CONFIRMED) |
| **State** | Status와 동일하므로 사용 금지 (혼동 방지) |

## ✅ 금지되는 표현 vs 올바른 표현

### 데이터 관점의 표현 ❌

```
❌ "레코드 추가"
✓ "신청 기록 추가"

❌ "상태값 변경"
✓ "경기 상태를 RECRUITING에서 CONFIRMED로 변경"

❌ "데이터 조회"
✓ "경기 목록 조회" 또는 "내 신청 목록 보기"

❌ "값 계산"
✓ "신뢰도 점수 계산"

❌ "테이블에 INSERT"
✓ "신청 기록 생성" 또는 "신청함"
```

### 시스템 관점의 표현 ❌

```
❌ "이벤트 처리"
✓ "신청됨 이벤트 발생 → Host에게 알림 전송"

❌ "비동기 처리"
✓ "경기가 확정되는 동안에 모든 참여자에게 알림"

❌ "API 호출"
✓ "신청을 서버에 전송"

❌ "트랜잭션"
✓ "경기가 확정될 때 모든 참여자에게 알림"
```

### 올바른 표현 ✓

```
✓ "User가 Match에 신청한다"
✓ "Participation이 생성된다"
✓ "Match 상태가 RECRUITING에서 CONFIRMED로 변경된다"
✓ "ReputationScore가 자동으로 계산된다"
✓ "Host에게 알림이 전송된다"
✓ "신청자가 4명이 되면 경기가 자동으로 확정된다"
✓ "노쇼하면 신뢰도가 -0.5점 내려간다"
```

## 📚 Context별 용어 빠른 참조

### Match Context (경기)
- 경기, 주최자, 참여자, 코트, 모집 중, 확정됨, 완료됨, 취소됨

### Participation Context (신청)
- 신청, 신청자, 신청함, 확정됨, 완료됨, 노쇼, 취소됨

### User Context (사용자)
- 사용자, 신뢰도, 평점, 리뷰, 노쇼, 신뢰도 점수

### Location Context (장소)
- 코트, 위치, 위도, 경도, 시설 정보

### Notification Context (알림)
- 알림, 이벤트, 채널, 푸시, 문자, 이메일, 앱 내
