# Business Logic Specification

## 개요
본 문서는 비즈니스 규칙과 도메인 로직을 정의합니다. 실제 프로젝트 개발 시 상세 규칙을 작성합니다.

## [도메인명] 비즈니스 규칙

### 개요
도메인의 핵심 개념과 목적 설명

### 주요 유스케이스
1. **[유스케이스명]**
   - **목적**: 유스케이스 목적
   - **액터**: 누가 사용하는가
   - **전제조건**: 실행 전 충족되어야 할 조건
   - **흐름**:
     1. 단계 1
     2. 단계 2
     3. 단계 3
   - **후처리**: 실행 후 상태 변화
   - **예외 케이스**:
     - 예외 상황 1
     - 예외 상황 2

### 비즈니스 규칙 (Business Rules)
1. **[규칙명]**
   - **설명**: 규칙 상세 설명
   - **조건**: 규칙이 적용되는 조건
   - **결과**: 규칙 적용 시 결과
   - **예외**: 규칙 위반 시 발생 예외

### 도메인 이벤트
1. **[이벤트명]**
   - **발생 시점**: 이벤트 발생 조건
   - **데이터**: 이벤트에 포함되는 데이터
   - **구독자**: 이벤트를 처리하는 주체

### 제약사항 (Constraints)
- 제약사항 1
- 제약사항 2

---

## 작성 가이드
1. 도메인 전문가와 협의하여 작성
2. 비즈니스 용어는 유비쿼터스 언어 사용
3. 규칙은 구체적이고 명확하게 작성
4. 예외 케이스는 반드시 명시
5. 이벤트 기반 아키텍처 사용 시 도메인 이벤트 정의
