# 클로드 사용 요강

- / command로 mcp, hook, 사용량 조회, 현재 사용 모델, context window, context window clear, 이전 내용 기억

## work flow

1. 요구사항을 어떻게 구현할 지, 문제를 파악한다.
2. 파악한 문제 시퀀스를 마크다운 문서로 그린다.
3. 마크다운 문서를 ai와 함께 수정 작업을 한다. 트레이드오프와 고민 내용을 문서에 작성하도록 한다.
4. 최종적으로 개발 절차를 정한다. 정해진 절차 문서대로 개발한다.

## 문서
- ai한테만 보여주는 것이 아니라 사람이 이해할 수 있는 문서로 번역한다.
- 작성된 문서의 정확하게 경로를 지정하고 알려줘야 한다. ex) /docs/spec/schema.md를 읽고 도메인, 엔티티를 작성해줘
- 이미지 -> 시퀀스 -> 마크다운 문서 -> 구현 -> 테스트

## ps
