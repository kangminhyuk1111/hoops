# Skills Trigger Guide

## Quick Reference

| Skill | Trigger |
|-------|---------|
| `/architecture-patterns` | 패키지 구조 (package structure), 도메인 설계 (domain design), 헥사고날 (hexagonal), 리팩토링 (refactoring) |
| `/create-pull-request` | PR 생성 (create PR), 풀 리퀘스트 (pull request), 머지 (merge) |
| `/debugging-strategies` | 버그 (bug), 에러 (error), 디버깅 (debugging), 성능 이슈 (performance), 메모리 릭 (memory leak) |
| `/code-review-excellence` | 코드 리뷰 (code review), PR 리뷰 (PR review), 피드백 (feedback) |
| `/tdd-workflow` | 기능 구현 (implement feature), 테스트 작성 (write test), 버그 수정 (fix bug), 시나리오 (scenario) |

## Detailed Triggers

### /architecture-patterns
- 새 도메인 생성 (new domain creation)
- 패키지 구조 질문 (package structure question)
- 헥사고날/클린 아키텍처 (hexagonal/clean architecture)
- DDD 패턴 (entity, VO, repository, port)
- 계층 의존성 (layer dependency)

### /create-pull-request
- PR 만들어줘 (create PR)
- 풀 리퀘스트 생성 (create pull request)
- 변경사항 제출 (submit changes)
- 기능 구현 완료 후 (after feature implementation)

### /debugging-strategies
- 테스트 실패 (test failure)
- 런타임 에러 (runtime error)
- 성능 이슈 (performance issue)
- 스택 트레이스 분석 (stack trace analysis)
- 왜 안 돼? (why not working?)

### /code-review-excellence
- 코드 리뷰해줘 (review code)
- PR 봐줘 (check PR)
- 코드 품질 피드백 (code quality feedback)

### /tdd-workflow
- 기능 구현해줘 (implement feature)
- 버그 수정해줘 (fix bug)
- 새 기능 요청 (new feature request)
- 코드 작성 전 (before writing code)
